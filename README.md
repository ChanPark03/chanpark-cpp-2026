## 학습기록

### 2026-04-06

- 메모리 주소 에는 1바이트 단위로 번호가 붙어있다. 1바이트 = 8비트

- 컴퓨터는 이동작을 한다. 1. 명령어 호출(fetch) 2. 명령어 해석(decode) 3. 명령어 실행(execute) 4. 결과 저장(store)
- CPU의 클럭 속도는 1초에 몇번의 명령어를 처리할 수 있는지를 나타낸다. 1Hz = 1초에 1회, 1GHz = 1초에 10억회.
- 파이프라인 : 명령어의 각 단계가 동시에 처리되는 방식. 예를 들어, 명령어 A가 실행 단계에 있을 때, 명령어 B는 해석 단계에 있고, 명령어 C는 호출 단계에 있을 수 있다. 이렇게 하면 CPU의 효율이 높아진다.
- 명령어 세트 : CPU가 이해하고 실행할 수 있는 명령어들의 집합. 예를 들어, x86, ARM 등이 있다.
- 레지스터 : CPU 내부에 있는 작은 저장 공간으로, 데이터를 빠르게 읽고 쓸 수 있다. 일반적으로 32비트 또는 64비트 크기를 가진다.
- 메모리 계층 : CPU 레지스터 > 캐시 > 메인 메모리 > 보조 저장장치
- 캐시 : CPU와 메인 메모리 사이에 위치한 고속 메모리로, 자주 사용되는 데이터를 저장하여 CPU의 접근 속도를 높인다. L1, L2, L3 캐시가 있다.
- 가상 메모리 : 운영체제가 물리적 메모리를 효율적으로 관리하기 위해 사용하는 기술로, 프로그램이 사용할 수 있는 메모리 공간을 물리적 메모리보다 크게 보이게 한다. 
-ROM(Read-Only Memory) : 읽기 전용 메모리로, 컴퓨터가 부팅될 때 필요한 펌웨어나 BIOS 등이 저장되어 있다. 일반적으로 변경할 수 없다.
-RAM(Random Access Memory) : 임의 접근 메모리로, 컴퓨터가 실행 중인 프로그램과 데이터를 저장하는 데 사용된다. 전원이 꺼지면 데이터가 사라진다.
- CPU는 명령어를 실행하기 위해 메모리에서 데이터를 읽고 쓰는데, 이 과정에서 메모리 계층이 중요한 역할을 한다. 레지스터는 가장 빠른 접근 속도를 제공하지만 용량이 작고, 캐시는 중간 정도의 속도와 용량을 제공하며, 메인 메모리는 가장 느리지만 가장 큰 용량을 제공한다. 가상 메모리는 프로그램이 사용할 수 있는 메모리 공간을 물리적 메모리보다 크게 보이게 하여, 프로그램이 필요한 만큼의 메모리를 사용할 수 있도록 한다.
