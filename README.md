## 학습 기록

### 2026-04-06

- 메모리 주소에는 1바이트 단위로 번호가 붙어 있다. 1바이트 = 8비트
- 컴퓨터는 다음과 같은 동작을 한다.
  1. 명령어 호출(fetch)
  2. 명령어 해석(decode)
  3. 명령어 실행(execute)
  4. 결과 저장(store)
- CPU의 클럭 속도는 1초에 몇 번의 명령어를 처리할 수 있는지를 나타낸다. 1Hz = 1초에 1회, 1GHz = 1초에 10억 회
- 파이프라인: 명령어의 각 단계가 동시에 처리되는 방식. 예를 들어, 명령어 A가 실행 단계에 있을 때 명령어 B는 해석 단계에 있고, 명령어 C는 호출 단계에 있을 수 있다.
- 명령어 세트: CPU가 이해하고 실행할 수 있는 명령어들의 집합. 예를 들어, x86, ARM 등이 있다.
- 레지스터: CPU 내부에 있는 작은 저장 공간으로, 데이터를 빠르게 읽고 쓸 수 있다. 일반적으로 32비트 또는 64비트 크기를 가진다.
- 메모리 계층: CPU 레지스터 > 캐시 > 메인 메모리 > 보조 저장장치
- 캐시: CPU와 메인 메모리 사이에 위치한 고속 메모리로, 자주 사용되는 데이터를 저장하여 CPU의 접근 속도를 높인다. L1, L2, L3 캐시가 있다.
- 가상 메모리: 운영체제가 물리적 메모리를 효율적으로 관리하기 위해 사용하는 기술로, 프로그램이 사용할 수 있는 메모리 공간을 물리적 메모리보다 크게 보이게 한다.
- ROM(Read-Only Memory): 읽기 전용 메모리로, 컴퓨터가 부팅될 때 필요한 펌웨어나 BIOS 등이 저장되어 있다. 일반적으로 변경할 수 없다.
- RAM(Random Access Memory): 임의 접근 메모리로, 컴퓨터가 실행 중인 프로그램과 데이터를 저장하는 데 사용된다. 전원이 꺼지면 데이터가 사라진다.
- CPU는 명령어를 실행하기 위해 메모리에서 데이터를 읽고 쓰는데, 이 과정에서 메모리 계층이 중요한 역할을 한다. 레지스터는 가장 빠른 접근 속도를 제공하지만 용량이 작고, 캐시는 중간 정도의 속도와 용량을 제공하며, 메인 메모리는 가장 느리지만 가장 큰 용량을 제공한다.

### 2026-04-07

- C++의 표준 라이브러리: C++에서 제공하는 다양한 기능과 자료구조를 포함하는 라이브러리다. 예를 들어, `<iostream>`은 입출력 스트림을 제공하고, `<vector>`는 동적 배열을 제공한다.
- `namespace`: C++에서 이름 충돌을 방지하기 위해 사용하는 기능이다. 예를 들어, `namespace std`는 표준 라이브러리의 이름을 포함하는 네임스페이스이다.
- `std::cout`: C++에서 표준 출력 스트림을 나타내는 객체로, 콘솔에 텍스트를 출력하는 데 사용된다.

```cpp
std::cout << "Hello, World!" << std::endl;
```

- `using namespace std;`를 사용하면 `std::cout` 대신 `cout`을 사용할 수 있다.

```cpp
using namespace std;
cout << "Hello, World!" << endl;
```

### 2026-04-08

- `cin`: C++에서 표준 입력 스트림을 나타내는 객체로, 콘솔에서 사용자 입력을 받는 데 사용된다.

```cpp
int x;
std::cin >> x;
```

- C언어와 C++의 차이점: C는 절차적 프로그래밍 언어로, 함수 중심의 구조를 가지고 있다. 반면 C++는 객체 지향 프로그래밍 언어로, 클래스와 객체를 사용하여 데이터를 캡슐화하고 재사용성을 높인다.
- 풀업 저항: 디지털 회로에서 입력 핀을 기본적으로 높은 전압 상태로 유지하기 위해 사용되는 저항이다.
- 풀다운 저항: 디지털 회로에서 입력 핀을 기본적으로 낮은 전압 상태로 유지하기 위해 사용되는 저항이다.
- 트랜지스터: 디지털 회로에서 스위치 역할을 하며 전류를 증폭하거나 스위칭하는 기능을 가진다.
- 저항: 전류의 흐름을 제한하는 전자 부품으로, 단위는 옴(Ω)이다.
- 콘덴서: 전하를 저장하는 전자 부품으로, 단위는 패럿(F)이다.
- 소자: 전자 회로에서 사용되는 기본적인 부품으로, 저항, 콘덴서, 트랜지스터 등이 있다.
- 디지털 회로: 이진 신호(0과 1)를 사용하여 정보를 처리하는 회로다.
- 임베디드 장치에서는 메모리가 소중하기 때문에 크기를 정확하게 예측 가능한 `uint16_t` 같은 자료형을 사용하는 것이 중요하다.

## C++ char 배열 개념 정리

### 1. 변수 스코프 (Scope)

- 변수는 선언된 `{}` 블록 안에서만 존재한다.
- `for (int i = 0; ...)`에서 `i`는 `for`문 밖에서 사용할 수 없다.
- 여러 `for`문에서 같은 변수를 쓰려면 바깥에서 선언해야 한다.

### 2. 배열 인덱스 범위

- `char arr[10]`의 사용 가능한 인덱스는 `0 ~ 9`이다.
- `arr[10]`은 존재하지 않는 칸이므로 범위 초과 버그가 발생한다.

### 3. 널 문자 `\0`

- 문자열의 끝을 알리는 신호다.
- char 배열은 항상 마지막에 `\0`이 필요하다.
- `while (arr[i] != '\0')`는 문자열 끝까지 순회하는 대표적인 패턴이다.

### 4. 배열 초기화

- `char arr[10] = {}`는 모든 칸이 `\0`으로 초기화된다.
- 조건 검사 시에는 복사할 배열을 기준으로 확인해야 한다.

### 5. 문자열 이어붙이기

- 첫 번째 배열 길이를 `i`에 저장한다.
- 두 번째 배열은 `fullname[i + j]` 위치부터 채운다.

```cpp
for (i = 0; i < 10; i++) {
    if (firstname[i] != '\0') {
        fullname[i] = firstname[i];
    } else {
        fullname[i] = '\0';
        break;
    }
}

for (j = 0; j < 10; j++) {
    if (lastname[j] != '\0') {
        fullname[i + j] = lastname[j];
    } else {
        fullname[i + j] = '\0';
        break;
    }
}
```
